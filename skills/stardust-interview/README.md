# Stardust Interview Skill

`stardust-interview` 用于 Stardust Inc. / PreSeen Inc. 的候选人面试准备、面试复盘、结构化面评撰写、小青面试系统提交和招聘群同步。

## 适用场景

用户提到以下内容时应使用本 skill：

- 星尘、PreSeen、小青面试系统、`xiaoqing_interview`
- 候选人分析、面试建议、面试问题
- AI 听记面试记录、结构化面评、提交面评
- 一面、二面、三面复盘
- 录用建议或招聘群面评同步

## 事实源顺序

1. 小青 MCP 是候选人业务事实源：候选人、岗位画像、简历、历史面试记录、评分项和提交目标。
2. DWS 是钉钉 AI 听记事实源：摘要、转写、会议时间、参会人和 task ID。
3. 本地文件、SQLite、源码或浏览器页面只用于排错，不能替代小青业务查询。

## 基本原则

- 区分材料事实、面试官判断和 agent 分析。
- 不被 AI 摘要带偏；摘要和转写冲突时，以转写、面试官判断和岗位画像为准。
- 默认严判，不把“参与过项目”直接当成能力证明。
- 逐题看回答质量，检查候选人是否正面回答、给出案例、说明指标、决策、取舍和结果。
- 写入小青前必须给用户看结构化面评草案并得到确认。
- 提交小青面评必须先 dry run，再用相同 payload 正式提交。

## 工作流程

面试准备：

1. 用小青搜索并锁定候选人。
2. 读取完整候选人上下文和岗位画像。
3. 必要时下载简历、附件和历史面试材料。
4. 如果有 AI 听记，读取摘要和完整转写。
5. 对历史轮次做“问题-回答-判断”审计。
6. 输出候选人定位、匹配程度、风险、薪资匹配、文化匹配和本轮追问。
7. 如运行环境支持，设置 2 小时后自动复盘任务。

面试后复盘：

1. 刷新小青上下文和目标面试轮次 revision。
2. 读取 DWS AI 听记摘要和转写。
3. 按岗位关键项、证据链、风险点和面试官判断写结构化面评草案。
4. 用户确认后执行 `upload_interview_result(dry_run=true)`。
5. dry run 通过后正式提交。
6. 按需同步到招聘群，并说明与前轮判断的差异。

## 输出要求

面试准备至少包含：

- 候选人定位
- 岗位关键要求
- 人员匹配程度
- 潜在风险
- 薪资匹配度
- 人员潜力评估
- 企业文化匹配
- 前面轮次主要问题和判断结论
- 本轮不再重复的问题
- 仍未闭环的关键判断
- 本轮递进追问和通过/风险标准
