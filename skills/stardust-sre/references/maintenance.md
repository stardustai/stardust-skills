# 维护、例外与质量指标

## 所有权与复核

- 当前内部基线版本：`1.0.0`。
- 控制负责人：于海龙；必须指定至少一名正式授权的 SRE 代理人，避免审批单点。
- 每季度以及 Kubernetes/Docker/CI 平台重大升级、生产事故或审计发现后复核。
- 变更 Skill 时运行 `scripts/self_test.py`，再用至少一个真实项目执行生成和验证；
  重大变更先试点，保留上一可用版本和回滚方法。

## 例外记录

无法满足阻断规则时，不得直接删除规则。例外至少记录：

- 规则编号、项目、环境和资源。
- 技术原因及风险评估。
- 负责人、审批人和批准时间。
- 补偿控制、验证证据和监控。
- 到期日期及彻底修复计划。

到期例外必须重新阻断，不能自动续期。涉及明文凭据、特权容器、未审批生产域名、
缺少首次发布审批或无法恢复的数据库变更，不接受无期限例外。

## 满分验收指标

- Skill frontmatter、引用和 UI 元数据有效率：100%。
- 相同输入生成结果一致率：100%；服务删除后旧生成清单残留率：0%。
- 明文 Secret、latest、未固定 Digest、未审批域名和首次发布门禁检出率：100%。
- 工作负载类型和持久化判断正确率：至少 98%，所有推断均需人工确认。
- 严重安全控制漏检率：0；合法项目误阻断率目标不超过 5%。
- 每次发布保留工单、审批、CI、镜像供应链、域名和验证报告证据。
- 缺少工具或平台证据时必须明确降级，禁止错误宣称合规或生产就绪。

只有自动自测、真实项目回归、生产门禁和上述指标全部满足时，才按内部评分口径
记为 100 分；100 分表示通过定义的质量门槛，不表示生产风险为零。
