# 安全、供应链与合规标准

## Secret 与配置

- 非敏感配置使用 ConfigMap；敏感值使用外部 Secret Manager、External Secrets、
  SOPS 或 Sealed Secrets 等组织批准方案。普通 Kubernetes Secret 仅是 Base64，
  不得提交真实值。
- Secret 模板只能包含键名或明显占位符。启用静态加密、最小 RBAC、轮换、吊销、
  使用审计和短期凭据。TLS/密钥只读挂载并采用最严格的可行文件模式。

## 镜像供应链

- CI 构建镜像并生成 SBOM；扫描基础镜像、系统包和应用依赖。按组织 SLA 阻断
  未批准的严重/高危漏洞。
- 推送后解析不可变 Digest，验证来源与签名，再按 Digest 部署。Tag 仅用于流水线
  选择，不作为生产不可变性证据。
- 锁定依赖和基础镜像版本，保留构建来源、提交、构建者、扫描结果和制品签名。
  不在构建参数、层、缓存或日志中泄露凭据。

## 日志、指标与追踪

- 生产默认 INFO。每次 HTTP/gRPC/RPC 调用记录时间、服务、方法/操作、路由模板、
  状态、耗时、Trace/Correlation ID 和必要的非敏感主体标识。
- 默认禁止记录请求体、响应体、Authorization、Cookie、Token、Session ID、密码、
  密钥、支付数据和敏感个人信息。按数据分类使用字段白名单、集中脱敏和自动测试；
  不以黑名单作为唯一保护。
- 定义日志访问控制、加密、保留期限、数据驻留、完整性、告警和删除流程。高流量
  服务评估成本和性能，但不得丢失安全审计必需事件。

## 合规与运行证据

- 配置只是控制设计，不代表合规。将控制映射到组织适用的 SOC 2、ISO 27001、
  等保或隐私要求，并由控制负责人确认适用范围。
- 保存代码审查、变更工单、批准记录、流水线结果、镜像 Digest/SBOM/签名、漏洞
  例外、域名分配与 DNS/TLS 变更、部署记录、访问复核、备份和恢复测试、告警
  演练及事件记录。
- 明确补丁 SLA、漏洞管理、权限复核、密钥轮换、事件响应、业务连续性和供应商
  管理责任。所有例外记录负责人、理由、补偿控制和到期日期。
- 未完成威胁建模、数据分类、集群策略确认、备份恢复测试或首次发布审批时，明确
  标记为生产阻断项，不得用“清单校验通过”替代运营证据。
