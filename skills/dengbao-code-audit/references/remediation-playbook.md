# 整改建议库

## 登录认证

整改：
- 统一后端认证中间件/Guard，所有非公开 API 默认需要认证。
- 密码最小长度不少于 8 位，包含复杂度要求；禁止常见弱口令。
- 登录失败增加锁定、验证码或限流。
- 管理员启用 MFA、IP 白名单或二次验证。
- 密码使用 bcrypt/argon2/PBKDF2 加盐哈希。

验收：
- 未登录访问受保护接口返回 401/403。
- 连续失败登录触发锁定或限流。
- 数据库中不存在明文密码。

## 权限控制

整改：
- 后端按角色、权限、租户、组织、资源 owner 校验访问。
- API 不信任请求体中的 userId、tenantId、role；从认证上下文获取。
- 管理操作增加审计和必要二次确认。
- 禁用/离职账号在认证阶段直接拒绝。

验收：
- 普通用户不能调用管理员接口。
- A 用户不能访问 B 用户或其他租户数据。
- 前端隐藏按钮后，直接调 API 仍会被后端拦截。

## 安全审计

整改：
- 对登录、用户管理、权限变更、数据导出、删除、配置变更、发布操作记录审计日志。
- 审计日志包含 actor、IP、时间、操作、对象、结果、失败原因、requestId。
- 普通日志增加敏感字段脱敏。
- 审计日志独立存储并限制删除权限。

验收：
- 能按用户、时间、对象追溯关键操作。
- 日志中不出现明文密码、token、身份证、完整手机号、私钥。

## 数据安全

整改：
- 建立敏感字段清单并在 DTO/Serializer/API response 层脱敏。
- 密钥和连接串进入密钥管理或环境变量，不提交代码仓库。
- 备份加密并限制访问。
- 生产数据进入测试环境前必须脱敏。

验收：
- API 响应、导出、日志中敏感字段按规则脱敏。
- 仓库中不存在真实密钥、生产连接串和私钥。

## 接口安全

整改：
- SQL/NoSQL 使用参数化查询或 ORM 安全 API。
- 文件上传校验后缀、MIME、大小、内容，并重命名存储到非执行目录。
- 服务端请求外部 URL 使用白名单、协议限制、内网地址拦截。
- 禁止用户输入进入 shell 命令、模板、路径拼接。
- 前端渲染 HTML 前使用可信消毒库，优先避免直接渲染 HTML。

验收：
- 注入测试、越权测试、上传测试、SSRF 测试均被拦截。
- 静态扫描不再命中对应高危模式，或有明确安全封装。

## 传输安全

整改：
- 公网统一 HTTPS，HTTP 跳转 HTTPS。
- Cookie 设置 Secure、HttpOnly、SameSite。
- 证书、HSTS、TLS 版本由网关/Nginx/Ingress 统一配置。
- 内部传输敏感数据时使用加密链路或专网隔离。

验收：
- 登录和 API 请求均为 HTTPS。
- 浏览器 Cookie 安全属性完整。

## 运维暴露面

整改：
- 数据库、缓存、MQ、SSH、监控、管理后台不得直接公网开放。
- 管理端仅通过 VPN、堡垒机、内网或白名单访问。
- 生产关闭 Swagger、debug、Actuator 敏感端点或加认证。
- 容器不使用 root，最小化镜像权限。

验收：
- 外网无法访问内部服务端口和管理端点。
- 端口暴露清单和资产清单一致。

## 备份恢复

整改：
- 明确备份范围、频率、保留周期、加密策略、恢复流程。
- 定期恢复演练并记录结果。
- 备份文件不放在 Web 可访问目录。

验收：
- 能从备份恢复到指定时间点。
- 有最近一次恢复演练记录。

## 发布变更

整改：
- 生产发布需要审批、记录、回滚方案和发布后验证。
- 数据库脚本必须有回滚或修正方案。
- CI/CD 保护生产分支和生产环境变量。

验收：
- 任一发布可追溯到审批、执行人、版本、变更内容、验证结果。

## 文档材料

整改：
- 补齐系统边界、网络拓扑、资产清单、账号清单、权限清单、数据流向。
- 补齐第三方接口、敏感数据清单、备份策略、应急预案、发布流程。
- 云上系统收集云厂商等保和安全能力证明。

验收：
- 测评人员可基于材料理解系统边界、数据流和责任边界。
