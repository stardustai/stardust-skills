# 等保代码安全检查清单

用于源码、配置、部署文件和项目文档层面的等保三级准备检查。正式测评还需要制度、运维记录、现场环境和第三方证明材料。

## 1. 登录认证

检查目标：确认用户和管理员身份鉴别足够可靠。

重点检查：
- 是否存在统一登录入口和后端认证中间件。
- 密码复杂度、最小长度、历史密码、初始密码修改策略。
- 登录失败锁定、验证码、限流、防爆破。
- 管理员或高权限账号是否支持双因素认证或更强校验。
- Session/JWT 过期时间、刷新机制、退出登录失效机制。
- 密码是否使用 bcrypt、argon2、PBKDF2 等慢哈希并加盐存储。

常见问题：
- 默认密码、弱口令、硬编码账号。
- 只在前端控制登录状态，后端接口未鉴权。
- JWT 密钥硬编码或过期时间过长。
- 管理后台没有 MFA、IP 限制或二次确认。

证据：
- 登录 Controller/API、认证中间件、用户模型、密码加密函数、Session/JWT 配置。

## 2. 权限控制

检查目标：确认访问控制在后端强制执行，且满足最小权限。

重点检查：
- API 是否统一经过鉴权和权限校验。
- 角色、权限、菜单、数据权限的关系是否清晰。
- 管理员、普通用户、租户、组织、项目之间是否做隔离。
- 是否存在水平越权、垂直越权、IDOR。
- 离职账号和禁用账号是否会被拒绝访问。

常见问题：
- 只隐藏前端按钮，不校验后端接口。
- API 直接按传入 userId、tenantId、orgId 查询数据。
- 超级管理员权限过大且无审计。
- permitAll、anonymous、skipAuth、publicRoute 范围过宽。

证据：
- Router/Controller、Guard/Middleware/Interceptor、RBAC/ABAC 实现、SQL 查询条件、租户过滤逻辑。

## 3. 安全审计

检查目标：确认关键操作可追踪、可归责、可保全。

重点检查：
- 登录、登出、失败登录、权限变更、用户管理、数据导出、删除、审批、配置变更、发布操作是否记录审计日志。
- 日志是否包含用户 ID、用户名、IP、时间、操作、对象、结果、失败原因、请求 ID。
- 审计日志是否防篡改、定期归档、保留周期明确。
- 普通业务日志是否避免输出密码、token、身份证、手机号、密钥等敏感信息。

常见问题：
- 只有普通运行日志，没有审计日志。
- 日志无法定位具体用户或操作对象。
- 日志可被应用管理员随意删除。
- 敏感数据直接打印到日志。

证据：
- AuditLog 模块、日志拦截器、操作日志表、日志配置、日志脱敏工具。

## 4. 数据安全

检查目标：确认敏感数据在存储、传输、展示、日志、导出、备份中受保护。

重点检查：
- 敏感字段识别：密码、token、身份证、手机号、银行卡、地址、个人信息、业务机密。
- 密码和密钥不可明文存储。
- 敏感字段是否加密、脱敏或按权限展示。
- 数据导出是否审批、脱敏、限量、留痕。
- 备份是否加密、访问受控、可恢复。

常见问题：
- 明文手机号、身份证、token 全量返回前端。
- 生产数据复制到测试环境。
- 备份文件、SQL dump、Excel 导出可被直接下载。
- `.env`、配置文件、日志中出现真实密钥。

证据：
- Entity/Model/DTO、序列化逻辑、导出接口、备份脚本、日志脱敏函数、配置文件。

## 5. 接口安全

检查目标：识别常见应用漏洞和输入处理缺陷。

重点检查：
- SQL/NoSQL 注入：是否参数化查询，是否拼接 SQL。
- XSS：是否输出不可信 HTML，是否使用危险渲染 API。
- 文件上传：后缀、MIME、大小、内容、存储路径、访问权限。
- 命令执行：是否用用户输入拼接 shell 命令。
- SSRF：是否允许用户传入 URL 并由服务端请求。
- 路径穿越：是否使用用户输入拼接文件路径。
- CSRF、CORS、安全响应头、Cookie Secure/HttpOnly/SameSite。

常见问题：
- `execute(sql + keyword)`、`rawQuery` 拼接参数。
- `dangerouslySetInnerHTML`、`v-html` 渲染未清洗内容。
- 上传文件可覆盖路径或上传可执行文件。
- `requests.get(url)`、`axios.get(req.query.url)` 无白名单。

证据：
- Controller/Service/DAO、上传处理、模板渲染、HTTP 客户端调用、前端渲染组件。

## 6. 传输安全

检查目标：确认公网和敏感内部通信使用加密传输。

重点检查：
- 公网页面和 API 是否强制 HTTPS。
- Nginx/Ingress/网关是否配置 TLS、HSTS、证书。
- Cookie 是否配置 Secure、HttpOnly、SameSite。
- 内部调用是否明文传递敏感信息。
- 证书过期、弱 TLS、HTTP 回源需待运维确认。

常见问题：
- 前端配置写死 `http://` API。
- 登录、文件下载、管理接口非 HTTPS。
- 内部服务间直接明文传输 token 或敏感数据。

证据：
- 前端环境变量、API client、Nginx/Ingress 配置、网关配置、Cookie/session 配置。

## 7. 运维暴露面

检查目标：确认生产环境不暴露高危管理端口和内部服务。

重点检查：
- Docker Compose、Kubernetes Service/Ingress、Nginx、Terraform、Ansible、systemd。
- SSH、数据库、Redis、RabbitMQ、Kafka、Elasticsearch、Prometheus、Grafana、Adminer、Swagger、Actuator 是否公网开放。
- debug、dev、test、mock 功能是否在生产关闭。

常见问题：
- `0.0.0.0:3306`、`6379:6379`、`22:22`、`15672:15672` 暴露。
- Swagger、Actuator、Django admin、Spring Boot actuator 无认证。
- 测试环境连接生产数据库。

证据：
- `docker-compose.yml`、`Dockerfile`、K8s yaml、Nginx conf、环境变量、启动脚本。

## 8. 备份恢复

检查目标：确认备份存在、受保护、可恢复。

重点检查：
- 是否有数据库、对象存储、配置文件、关键业务文件备份策略。
- 是否有恢复脚本、恢复文档、恢复演练记录。
- 备份是否加密、异地/离线保存、访问受控。
- 代码中是否存在导出/备份接口或脚本，需要权限和审计。

常见问题：
- 只有备份任务，没有恢复演练。
- 备份文件和数据库导出文件放在 Web 可访问目录。
- 备份文件未加密或包含明文敏感数据。

证据：
- backup/restore 脚本、cron、CI job、运维文档、对象存储配置。

## 9. 发布变更

检查目标：确认上线有审批、记录、回滚和验证。

重点检查：
- CI/CD 是否有审批或保护分支。
- 数据库脚本、配置变更、灰度发布、回滚脚本是否留痕。
- 发布后验证清单和关键指标是否明确。
- 紧急发布是否有事后补录机制。

常见问题：
- 直接手工改生产配置。
- 数据库脚本无回滚方案。
- 发布失败无记录、无责任人、无验证清单。

证据：
- CI 配置、部署脚本、migration、release 文档、变更记录模板。

## 10. 文档材料

检查目标：确认正式测评需要的边界和材料可准备。

重点检查：
- 系统边界、网络拓扑、资产清单、账号清单、角色权限清单。
- 数据流向、敏感数据清单、第三方接口清单。
- 安全制度、应急预案、备份策略、发布变更流程。
- 云资源和云厂商等保证明材料。

常见问题：
- 源码有系统实现，但没有系统边界和数据流向说明。
- 无资产清单、账号清单、权限清单。
- 无安全制度和运维记录。

证据：
- README、docs、architecture、deployment、security、operations、接口文档。
