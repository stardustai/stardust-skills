## 后续生成测试用例时的强约束

1. 工作池或任务流转需求：默认覆盖标注池、检查池、抽查池、完成池，以及退回、驳回、暂停、释放、回收、完成和错标状态。
2. 标注工具需求：默认覆盖操作项、属性项、实例面板、画布、快捷键、撤销/重做、多视图、连续帧、映射、错标、导出结果和多模态兼容。
3. 任务列表、统计或导出需求：默认覆盖筛选条件组合、分页/跨页、数据总量、导出字段顺序、空数据、大数据量、异步处理、权限越权和页面数据一致性。
4. 数据管理或数据探索需求：默认覆盖数据集类型、数据格式、导入路径、数据源权限、版本、切片、送标、数据流、数据权限隔离和导出格式。
5. 组织、用户、角色或登录需求：默认覆盖 SaaS/私有化差异、租户/组织隔离、团队角色、用户状态、账号验证、允许登录时间、邀请、注销和 API 越权。
6. 算法需求：默认覆盖算法创建、编辑、删除、查询、关联项目、操作项映射、参数配置、调用日志、失败提示、标注页按钮展示和算法结果回写。
7. 阶段版本需求：出现“一期/二期/三期/V3.x/废弃/待讨论/未实现”等标记时，必须优先判断版本覆盖关系，不能把废弃或待讨论规则直接当作当前规则。
