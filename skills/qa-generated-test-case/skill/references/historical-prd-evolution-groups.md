# 历史 PRD 版本/阶段关系重点候选

这份文件用于提醒 Codex：当新需求提到一期、二期、后续修改、V3.x、废弃、待讨论、未实现时，要先检查历史版本关系，再生成测试用例。

完整本地文件：`memory/historical-prd-archive/evolution-groups.md`

## 重点候选关系

1. 标注工具
   相关文件包括 `3.0标注工具PRD.md` 与 `20220823 标注工具.md`。生成标注工具用例时要注意 3.0 与早期标注工具配置、任务列表、操作项能力之间的继承关系。

2. 算法管理
   相关文件包括 LLM 标注套件未实现中的 `算法管理.md` 与 `算法接入/算法管理二期.md`。生成算法用例时要区分未实现设想与已实现二期规则。

3. 错误标签维度上增加可申诉状态
   同一主题出现在标注工作和项目管理套件/工作流下。生成错标、错误标签、申诉状态相关用例时，要同时检查标注侧展示与项目管理侧配置/工作流影响。

4. Mega规则校验
   存在 `Mega规则校验-一期.md` 与 `Mega规则校验-二期.md`。生成任务列表或规则校验用例时，要优先识别二期是否覆盖一期规则。

5. 数据流转
   存在 `20220706 数据流转PRD.md` 与 `20220718 数据流转PRD.md`。生成工作流/工作池/任务流转用例时，要检查后续版本对流转规则、状态和限制条件的修改。

6. 数据导入时做地线算法预处理
   同一主题出现在 Rosetta 项目管理数据集和多模态数据管理数据集管理下。生成数据导入/算法预处理用例时，要覆盖 Rosetta 与 MS/多模态数据管理的兼容关系。

7. 模型评估指标丰富
   存在一期、二期，并且目录标记为待讨论。生成模型评估用例时，要把待讨论内容作为待确认，不要直接生成正式断言。

## 使用规则

1. 如果当前 PRD 文件名或正文出现“后续修改”“一期”“二期”“三期”“V3.x”，先搜索完整历史归档再生成用例。
2. 如果历史文件标记为“废弃”“待讨论”“未实现”，只能用来提出待确认问题或历史风险，不能直接当作当前上线规则。
3. 如果同一主题同时出现在多个产品域，例如 Rosetta 与多模态数据管理，必须覆盖跨产品域兼容、字段同步和旧数据回归。
