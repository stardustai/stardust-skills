# 任务列表页新增筛选，数据展示功能 - PRD 归一化

## 1. 背景与目标

当前任务列表页存在筛选能力不足、数量信息缺失、无法跨页选中、时间字段展示不够精准等问题。项目管理人员在任务数据量较大时，难以通过多维度条件快速定位目标任务，也无法直观看到筛选命中总量和已选中任务数量。

本次目标是提升项目管理人员在任务列表页上的查询、选择和批量操作效率。

## 2. 角色与对象

| 对象 | 说明 |
| --- | --- |
| 项目管理人员 | 主要使用者，具备项目管理下任务列表访问权限 |
| 任务列表页 | 位于项目管理模块，展示任务ID、标题、是否优先、数据集、当前操作人、当前操作团队、工作池、任务状态、驳回总次数、回收倒计时、分配时间、更新时间、操作等字段 |
| 任务数据 | 支持筛选、勾选、跨页累计选择、导出选中任务 |

## 3. 功能范围

### 3.1 多条件筛选

1. 点击【筛选】按钮后，在按钮下方弹出筛选条件面板。
2. 点击【+添加条件】后，默认出现“当前操作人”。
3. 用户可切换筛选维度，可选项包括但不限于任务ID、标题、是否优先、数据集、当前操作人、当前操作团队、工作池、任务状态等。
4. 不同筛选维度右侧展示对应控件，包括文本输入、下拉选择或时间选择。
5. 可添加多个筛选条件，多个条件默认按 AND 生效。
6. 输入筛选值或删除筛选条件后，即执行筛选。
7. 可点击【X】关闭某个条件标签。
8. 每一项具体筛选逻辑沿用平台当前逻辑。

### 3.2 筛选结果数与已选中数

1. 执行筛选后，【筛选】按钮右侧显示本次筛选命中的任务总数，例如“筛选 368”。
2. 无筛选条件时，不展示筛选结果数字。
3. 勾选任意任务后，【选择全部】按钮右侧显示“已选择 N 条 ×”。
4. 勾选、取消勾选、点击“×”清除后，已选中数量实时更新。
5. 筛选结果数和已选中数独立展示，不能互相覆盖或混算。

### 3.3 跨页选中与导出选中任务

1. 分页大小新增 500 条/页、1000 条/页。
2. 支持跨页选中，例如第 1 页选中 10 条、第 2 页选中 20 条，累计统计为 30 条。
3. 批量操作和导出选中任务时，操作范围应为用户实际选中的任务集合。

### 3.4 时间字段展示

1. 更新时间由相对时间改为绝对时间，格式为 `YYYY-MM-DD HH:mm:ss`。
2. 任务列表新增【分配时间】列。
3. 分配时间表示数据第一次流入标注池的时间。
4. 数据从未流入标注池时，分配时间展示为 `-`。
5. 分配时间只记录第一次流入时间，后续重新流转不覆盖。
6. 表格列顺序参考：任务ID、标题、是否优先、数据集、当前操作人、当前操作团队、工作池、任务状态、驳回总次数、回收倒计时、分配时间、更新时间、操作。

## 4. 业务规则和字段映射

| 规则编号 | 规则 |
| --- | --- |
| R1 | 多个筛选条件之间默认按 AND 生效 |
| R2 | 输入筛选值或删除条件后，列表结果和筛选结果数量均需刷新 |
| R3 | 筛选结果数量代表筛选命中总量，不是当前页数量 |
| R4 | 无筛选条件时不展示筛选结果数字 |
| R5 | 已选中数量代表用户主动勾选的任务数，和筛选结果数量独立 |
| R6 | 跨页选中时，不同页的选中任务需要累计统计 |
| R7 | 导出选中任务时，导出范围必须是实际选中集合 |
| R8 | 更新时间展示绝对时间，不再展示“X 天前” |
| R9 | 分配时间是任务第一次流入标注池的时间 |
| R10 | 未流入标注池的数据，分配时间展示为 `-` |
| R11 | 分配时间后续重新流转不覆盖 |

## 5. 历史业务记忆对照

本地业务记忆中已沉淀同名需求规则，内容与当前 PRD 基本一致：筛选面板位置、添加条件默认“当前操作人”、筛选维度、AND 关系、筛选数量展示命中总量、已选中数量展示、跨页选中、导出实际选中集合、更新时间绝对时间、分配时间首次流入标注池且不覆盖。

未发现与当前 PRD 冲突的历史规则。当前 PRD 仍优先。

## 6. 测试假设与待确认

1. “名称、单位多重维度筛选”在功能标题中出现，但详细筛选维度未直接出现“名称”和“单位”，需确认是否对应“标题”和“当前操作团队”或另有字段。
2. PRD 未说明是否允许添加重复筛选条件，需确认重复条件是否允许、是否覆盖、是否合并。
3. PRD 未说明筛选条件为空值时是否立即请求、忽略条件或提示必填。
4. PRD 未说明跨页选中后切换筛选条件时，已选集合是否清空或保留。
5. PRD 未说明无已选任务时“导出选中任务”入口是否禁用、隐藏或点击后提示。
6. PRD 未说明新增筛选和分配时间是否涉及新增权限 code，当前按已有任务列表权限处理。
7. PRD 未说明绝对时间的时区口径，当前仅按页面展示格式进行测试。
