任务列表页新增筛选，数据展示功能PRD
1. 基本信息
1.1 背景与目标
背景
当前任务列表页在数据筛选、任务统计和时间展示方面存在以下问题：
筛选能力不足：现有筛选功能仅支持单一维度搜索（搜索框输入），无法同时按名称、单位等多个维度组合筛选，导致数据量大时用户难以快速定位目标任务。
数量信息缺失：页面不展示当前筛选结果的任务总数，也不显示已选中任务的数量，用户无法直观感知操作范围。
跨页选中能力缺失：当前仅支持当前页内勾选，无法一次性选中全部数据（跨分页），导致批量操作需要逐页操作，效率低下。
时间字段不精准：更新时间以"X 天前"相对时间展示，无法得知具体操作时间；且缺少"下发时间"字段，无法追溯数据首次进入标注池的时间节点。
目标
通过本次功能优化，提升项目管理人员对任务列表的操作效率。
2. 功能设计
功能 1：支持名称、单位多重维度筛选
交互流程
用户在任务列表页点击【筛选】按钮，在【筛选】按钮下方弹出；
页面弹出筛选条件面板后，用户点击【+添加条件】，默认出现当前操作人
用户可点击当前操作人切换筛选条件，在面板内展示可选筛选维度，包括但不限于：任务ID、标题、是否优先、数据集、当前操作人、当前操作团队、工作池、任务状态等；
用户点击某一筛选维度后，右侧出现对应的输入或选择控件（文本输入 / 下拉选择 / 时间选择），可填写筛选值；
用户可添加多个筛选条件，条件之间默认为"且（AND）"关系；
当用户在输入框中输入具体值时/删除了某个筛选条件后，即可执行筛选；
用户可随时点击【X】关闭某个条件标签。
备注：
每一项具体的筛选逻辑根据平台目前逻辑进行。
功能 2：展示当前任务页面任务个数和已选中任务个数
交互逻辑
筛选结果数展示：执行筛选后，【筛选】按钮右侧显示本次筛选命中的任务总数，例如筛选结果共 368 条则显示 筛选 368；无筛选条件时不显示数字；
已选中数量展示：当用户勾选任意任务（单条或批量）后，【选择全部】按钮右侧出现已选中任务计数，格式为 已选择 N 条 ×，其中 × 为一键清除选中的操作入口；
数量实时更新：勾选或取消勾选任一条目时，已选中数量实时刷新；执行清除操作（点击 ×）后，隐藏该提示；
两个数量独立显示，互不干扰：筛选结果数代表当前列表命中总量，已选中数代表用户主动勾选的条目数。
功能 3：支持跨页选中（选中后支持导出选中的任务）
修改现有功能：
新增两个选项：500条/页，1000条/页
新增功能：
支持跨页选中，比如在页面1选中10条，在页面2选中20条，要求一共对选中的这30条数据进行数量统计和批量操作。
功能 4：任务更新时间、下发时间（第一次分配给标注池的时间）显示具体时间
更新时间字段改造
改动前：更新时间列显示相对时间，格式为"X 天前"（如：2 天前、5 天前）；
改动后：更新时间列显示具体的绝对时间，格式为 YYYY-MM-DD HH:mm:ss，（如：2025-11-02 12:00:00）；。
下发时间字段新增
在任务列表表格中新增【分配时间】列；
字段定义：数据第一次流入标注池的时间，即任务首次进入标注流程的时间节点；若数据从未流入标注池（如仅在分配池），该字段显示为 -；
字段格式与更新时间保持一致为 YYYY-MM-DD HH:mm:ss，（例如：2025-11-02 12:00:00）；
该字段仅记录第一次流入时间，后续重新流转不覆盖该值；
