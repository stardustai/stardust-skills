# license管理 - 测试设计

## 1. 测试目标

验证 Phoenix 平台 license 管理页、license 生成页、私有化部署授权到期处理链路符合 PRD 要求，重点覆盖入口变更、历史记录展示、搜索、字段映射、必填校验、复制、角色权限、授权恢复和异常 License 场景。

## 2. 覆盖分层

| 层级 | 覆盖重点 |
| --- | --- |
| 导航入口 | 侧边栏文案变更、license 管理页访问、+ 按钮跳转 |
| 列表展示 | 表头、字段映射、序号、状态、备注空值、License 内容、创建人 |
| 列表搜索 | 项目名称、类别、状态搜索，以及空结果和清空搜索 |
| 生成流程 | 标题面包屑、返回不保存、三类生成类型、字段增减、必填校验 |
| 兼容回归 | 独立标注工具旧流程、原私有化域名流程 |
| 私有化授权 | Admin 到期登录、申请码生成复制、License 写入、授权恢复 |
| 安全权限 | 非 Admin 到期拦截、非法 License 不恢复、环境不匹配不恢复 |

## 3. 角色-权限矩阵

| 角色 | 标准 Phoenix license 管理 | license 生成 | 私有化到期登录 | 到期后 license 页面操作 |
| --- | --- | --- | --- | --- |
| 项目经理 | 可查看历史记录、搜索、复制 | 可进入并生成 | 不适用 | 不适用 |
| Admin | 可按权限访问 | 可按权限生成 | 可正常登录并跳转 license 页面 | 可生成申请码、复制申请码、写入 License |
| 非 Admin 用户 | 按原有权限 | 按原有权限 | 不可正常使用完整系统 | 不可绕过授权到期限制 |

## 4. 状态流转表

| 状态对象 | 初始状态 | 触发动作 | 目标状态或结果 |
| --- | --- | --- | --- |
| 导航入口 | license生成 | 需求上线 | 侧边栏展示 license管理 |
| 生成页表单 | 已填写未提交 | 点击标题中的 license管理 返回 | 返回首页且表单内容不保存 |
| license 记录 | 新增提交成功 | 回到管理列表 | 列表新增记录，序号递增 |
| 私有化授权 | 规定期限外 | Admin 登录 | 登录成功并跳转 license 页面 |
| 私有化授权 | 规定期限外 | 非 Admin 登录 | 无法正常使用完整系统 |
| 申请码 | 未生成 | 点击生成申请码 | 系统按环境生成申请码 |
| License 写入 | 授权到期 | 写入合法 License 并确定 | 校验通过并恢复系统使用 |
| License 写入 | 授权到期 | 写入空或非法 License | 不恢复系统完整授权 |

## 5. 关键业务规则转测试规则

| 测试规则 | 来源规则 | 风险 |
| --- | --- | --- |
| 入口必须从 license生成 改为 license管理 | R1 | 用户仍进入旧页面或导航混乱 |
| + 按钮必须跳转 license 生成页 | R2 | 管理页无法创建新 license |
| 返回首页不保存未提交表单 | R4 | 脏数据残留或误生成 |
| 三类生成类型必须共存 | R5 | 新增申请码类型破坏旧类型 |
| 申请码和项目名称必填文案必须准确 | R8、R9 | 表单校验不一致影响验收 |
| 列表字段必须按类别正确映射 | R12、R13、R14 | 申请码和域名字段串值 |
| 复制必须复制完整 License 内容 | R15 | 复制截断内容导致客户不可用 |
| 到期后仅 Admin 可处理授权 | R16 | 普通用户绕过授权限制 |
| 合法 License 才能恢复系统 | R17 | 非法授权恢复带来安全风险 |

## 6. 高风险点

1. 入口只改了文案但路由、权限、面包屑仍指向旧 license 生成页面。
2. 管理列表表头与字段定义不一致，申请码、域名、生效时间、创建人、操作字段容易遗漏。
3. 申请码类型和域名类型字段映射互相串值，或者不适用字段未展示 `-`。
4. 状态判断规则存在申请时间和有效期口径歧义，容易导致有效和已过期判断错误。
5. 复制 License 内容时只复制页面可见截断内容，而不是完整内容。
6. 新增三类生成类型后，独立标注工具旧流程被破坏。
7. 授权到期后非 Admin 可绕过限制，或 Admin 不能进入 license 页面处理。
8. 空 License、非法 License、环境不匹配 License 被错误接受并恢复系统。

## 7. 测试数据建议

1. 准备项目经理账号、Admin 账号、非 Admin 普通账号。
2. 准备私有化部署-申请码记录，字段包含申请码、项目名称、席位数、标注数量、有效期、License 内容、备注。
3. 准备私有化部署-域名记录，字段包含域名、项目名称、席位数、标注数量、有效期、License 内容、备注。
4. 准备备注为空和备注有值的记录。
5. 准备有效和已过期两类状态记录。
6. 准备长 License 内容，用于验证复制完整性。
7. 准备授权到期的私有化部署环境、合法 License、空 License、非法 License、环境不匹配 License。

## 8. 待确认问题

1. license 管理列表最终字段和列顺序，以表头清单为准还是以字段说明为准。
2. 状态判断到底依据申请时间、有效期、到期时间，还是后端授权状态。
3. 项目名称、类别、状态搜索是否支持组合搜索。
4. 复制 License 或申请码成功后的提示文案。
5. 非 Admin 授权到期登录的拦截文案和页面。
6. 空 License、非法 License、环境不匹配 License 的错误文案。
7. 私有化部署“规定期限外”的判断字段和时区。
