# license管理 - PRD 归一化

## 1. 背景与目标

当前 license 管理依赖项目经理个人表格记录，无法在 Phoenix 平台页面查看历史 license 记录。本次需求新增 license 管理页面，用于查看历史 license 记录，并调整 license 生成流程，补充私有化部署授权到期后的申请码生成、License 写入和系统恢复链路。

## 2. 角色与对象

| 角色或对象 | 说明 |
| --- | --- |
| 项目经理 | 在 Phoenix 平台查看历史 license 记录，进入 license 生成页，复制生成后的 License 内容 |
| Admin | 私有化部署项目授权到期后唯一可正常登录的角色，可生成申请码、复制申请码、写入 License |
| 非 Admin 用户 | 私有化部署项目授权到期后不能正常使用完整系统 |
| license 管理页 | 原侧边栏【license生成】入口调整为【license管理】，展示历史 license 表格 |
| license 生成页 | 从 license 管理页的 + 按钮进入，标题调整为 `license管理/license生成` |
| 私有化部署项目 license 页面 | 授权到期后 Admin 登录跳转的页面，支持申请码和 License 写入链路 |

## 3. 功能范围

### 3.1 license 管理页

1. 将侧边栏原【license生成】入口变更为【license管理】。
2. 【license管理】页面提供 `+` 按钮，点击后进入 license 生成页面。
3. 页面中央展示历史 license 表格。
4. PRD 明确表头包括：序号、类别、项目名称、状态、席位数、标注数量、申请时间、有效期、License 内容、备注。
5. PRD 同时描述了申请码、域名、生效时间、创建人、操作等字段，表头是否包含这些字段需确认。
6. 序号从 1 开始，每新增一个 license 记录序号 +1。
7. 项目名称支持搜索。
8. 类别支持搜索，类别包括私有化部署-申请码、私有化部署-域名。
9. 状态支持搜索，状态包括有效和已过期。
10. License 内容读取生成的 license 内容。
11. 操作区支持复制 License 内容。

### 3.2 license 列表字段规则

| 字段 | 规则 |
| --- | --- |
| 项目名称 | 私有化部署类型读取提交内容中的【项目名称】字段 |
| 类别 | 支持私有化部署-申请码、私有化部署-域名 |
| 申请码 | 私有化部署-申请码读取提交内容中的【申请码】字段；私有化部署-域名展示 `-` |
| 域名 | 私有化部署-申请码展示 `-`；私有化部署-域名读取提交内容中的【域名】字段 |
| 状态 | 展示有效或已过期；PRD 写为根据当前时间与申请时间对比判断 |
| 席位数 | 私有化部署类型读取提交内容中的【席位数】字段 |
| 标注数量 | 私有化部署类型读取提交内容中的【标注数量】字段 |
| 生效时间 | 私有化部署类型为申请提交当天 |
| 有效期 | 私有化部署类型读取提交内容中的【有效期】字段 |
| License 内容 | 私有化部署类型读取生成的 license 内容 |
| 创建人 | 私有化部署类型展示账号名称 |
| 备注 | 私有化部署类型读取提交内容中的【备注】字段；未填写展示 `-`；独立标注工具展示 `-` |

### 3.3 license 生成页

1. 页面标题由原 `license生成` 调整为 `license管理/license生成`。
2. 点击标题中的【license管理】可返回 license 管理首页。
3. 从生成页返回首页后，生成页中已填写但未提交的信息不保存。
4. 生成类型由原【私有化部署】【独立标注工具】调整为【私有化部署-申请码】【私有化部署-域名】【独立标注工具】。
5. 私有化部署-申请码在原有字段基础上新增申请码、项目名称、备注。
6. 申请码必填，不填写提交提示 `请输入邀请码`。
7. 项目名称必填，不填写提交提示 `请输入项目名称`。
8. 备注选填。
9. 私有化部署-域名在原有字段基础上新增项目名称、备注，规则同上。
10. 独立标注工具作为原有类型保留，不应被新增类型影响。

### 3.4 私有化部署项目授权到期链路

1. 私有化部署项目如果在规定期限外，只有 Admin 角色用户可以正常登录。
2. Admin 登录后直接跳转到 license 页面。
3. license 页面支持生成申请码。
4. 管理员点击【生成申请码】后，系统根据环境自动生成授权申请码。
5. 管理员点击【复制申请码】后，可复制申请码并提供给授权方。
6. 管理员将授权方返回的 License 内容粘贴到输入框，点击【确定】提交。
7. License 校验通过后，系统授权状态恢复正常，用户可继续使用完整系统功能。

## 4. 业务规则和字段映射

| 规则编号 | 规则 |
| --- | --- |
| R1 | Phoenix 侧边栏入口由【license生成】调整为【license管理】 |
| R2 | license 管理页通过 `+` 进入 license 生成页 |
| R3 | license 生成页标题展示为 `license管理/license生成` |
| R4 | 从 license 生成页返回 license 管理首页时，未提交表单内容不保存 |
| R5 | 生成类型包括私有化部署-申请码、私有化部署-域名、独立标注工具 |
| R6 | 私有化部署-申请码新增申请码、项目名称、备注 |
| R7 | 私有化部署-域名新增项目名称、备注 |
| R8 | 申请码为空提交提示 `请输入邀请码` |
| R9 | 项目名称为空提交提示 `请输入项目名称` |
| R10 | license 管理列表序号从 1 开始且新增记录递增 |
| R11 | 项目名称、类别、状态支持搜索 |
| R12 | 申请码类型记录中申请码取提交内容，域名展示 `-` |
| R13 | 域名类型记录中域名取提交内容，申请码展示 `-` |
| R14 | 备注为空时展示 `-` |
| R15 | 复制操作复制完整 License 内容 |
| R16 | 授权到期后仅 Admin 可正常登录并进入 license 页面 |
| R17 | 合法 License 写入并校验通过后恢复完整系统功能 |

## 5. 历史业务记忆对照

本地业务记忆中已有同名需求，核心规则与当前 PRD 一致：入口调整、生成类型拆分、申请码与项目名称必填、列表搜索、字段映射、复制 License、私有化授权到期 Admin 处理、合法 License 恢复系统。历史记忆额外提示状态判断口径存在“申请时间”和“有效期”的歧义，本次仍以当前 PRD 为准，并将该问题列入待确认。

## 6. 测试假设与待确认

1. license 管理列表 PRD 表头未列出申请码、域名、生效时间、创建人、操作，但正文定义了这些字段，需确认最终列表字段和顺序。
2. 状态判断规则写为“当前时间与申请时间对比”，但 license 通常与有效期相关，需确认有效和已过期的真实判断字段。
3. 私有化部署-申请码和私有化部署-域名中的“原有字段”未完整列出，需以现有页面为准回归原字段。
4. 类别搜索、状态搜索和项目名称搜索是单条件搜索还是支持组合搜索，PRD 未说明。
5. License 复制成功后是否需要 toast 提示，PRD 未说明。
6. 空 License、非法 License、环境不匹配 License 的错误提示文案未说明。
7. 授权到期后非 Admin 用户登录的拦截页面和提示文案未说明。
8. “规定期限外”的判断来源和时间口径未说明。
