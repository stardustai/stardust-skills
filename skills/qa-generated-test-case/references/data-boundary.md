# 数据边界

## 可随 skill 提交

1. 通用 workflow、提示结构、输出格式、质量评分规则。
2. 可复用脚本和轻量单测。
3. 合成样例或不包含真实客户/项目内容的小型 fixture。

## 不可随 skill 提交

1. 运行输出：`runs/`、CSV/XLSX、提取后的 PRD 文本、一次性分析草稿。
2. 历史 PRD 原文、完整历史 digest/index、业务记忆全集、客户或项目私有资料。
3. token、cookie、浏览器状态、本地配置、未确认脱敏的截图或日志。

## 历史材料的正确位置

历史 PRD 和业务记忆应放在以下任一位置：

1. Friday Memory 或其他经过授权的 memory/document store。
2. 明确标记为私有的数据目录。
3. 单独的数据包，并在 README 中说明访问权限、更新方式和敏感信息边界。

skill 只能读取检索后的 top N 片段。没有用户授权时，不要上传、复制或全文读取本地私有资料。
