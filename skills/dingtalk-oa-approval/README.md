# DingTalk OA Approval Skill

`dingtalk-oa-approval` 用于审阅 Derek 的钉钉 OA 审批、生成审批意见，并在满足条件时执行评论、同意或拒绝动作。

## 适用场景

用户提到以下事项时应使用本 skill：

- 钉钉审批、OA、待审批
- 同意、拒绝、退回、补材料
- 录用、离职、请假、出差、外出
- 财务付款、报销、合同、晋升、立项特批
- 定时检查新审批或超过 3 天的审批

## 基本原则

- 先审阅，后动作。
- 当前 `RUNNING` 任务必须属于 Derek，才继续完整审阅或执行动作。
- 审批详情、流水、附言、附件、链接文档、PDF、合同、面试记录、标准依据等材料必须读完。
- 材料缺失、无法访问、未集中归档到知识库项目文件夹时，不得建议通过。
- 评论不是同意或拒绝；当结论是补材料或说明风险时，优先评论清楚缺口。
- 只有“低风险 + 高确信度”且任务属于 Derek 时，才允许自动同意。

## 工具顺序

优先使用 `dws`，所有命令加 `--format json`：

```bash
dws oa approval list-pending --page 1 --size 30 --format json
dws oa approval detail --instance-id <processInstanceId> --format json
dws oa approval records --instance-id <processInstanceId> --format json
dws oa approval tasks --instance-id <processInstanceId> --format json
```

DWS 详情为空或字段丢失时，可在已有授权前提下用钉钉 OA OpenAPI 补读，但不得输出 token、AppKey、AppSecret、cookie 或临时签名 URL。

如果接口仍读不到全文，并且用户授权打开界面，可以使用钉钉客户端或 Chrome 补读。只读审阅时不要点击同意、拒绝或撤销。

## 审阅流程

1. 从待办列表提取审批标题、流程类型、实例 ID、状态和更新时间。
2. 用 tasks 确认当前 `RUNNING` 任务是否属于 Derek。
3. 读取审批详情、表单字段、附件和任务节点。
4. 读取审批流水和附言/评论。
5. 打开所有可访问的附件、链接文档、PDF 和相关材料。
6. 检查实质材料是否已归档到同一个知识库项目文件夹。
7. 对照制度、标准、预算、合同、面试记录或其他依据判断。
8. 给出风险等级、确信度、建议动作、核心理由、制度依据、关键风险和建议附言。
9. 在授权范围内执行评论、同意或拒绝。

## 风险和确信度

风险等级：

- 低风险：金额小或无金额影响，不形成重大合同、付款、录用、权限、数据或合规义务。
- 中风险：涉及预算、客户、差旅、轻量合同、项目资源或后续凭证闭环。
- 高风险：涉及录用、薪资、晋升、合同实质义务、授信、付款、数据隐私、权限或重大合规风险。

确信度：

- 高确信度：所有材料和依据读完，任务属于 Derek，无未解释矛盾。
- 中确信度：主体事实清楚，但有次要材料、归档或后续凭证待补。
- 低确信度：详情为空、材料不可读、标准缺失或评论中有未澄清信息。

## 输出要求

每条审批必须单独输出：

- 审批标题和实例 ID
- 当前任务是否属于 Derek
- 已读材料和未读到材料
- 风险等级
- 确信度
- 建议动作
- 核心理由
- 建议附言
